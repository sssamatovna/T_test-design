# Т1: Тестирование REST API: Получение статуса продукта 

Данный документ содержит план и тест-кейсы для метода:
`GET /products/{productID}/status?authToken=&recalculate=&owner=&region=`

---

## 1. План тестирования (Test Plan)

### 1.1 Область тестирования
Проверка функциональности эндпоинта получения статуса продукта, включая валидацию параметров авторизации, обработку обязательных/необязательных полей и корректность HTTP-ответов.

### 1.2 Применяемые техники тест-дизайна
* Анализ граничных значений (BVA): для проверки длины `authToken` (ровно 16 символов).
* Классы эквивалентности (EP): для типов символов в токене и состояний продукта.
* Таблица принятия решений: для проверки комбинаций необязательных параметров (`recalculate`, `owner`, `region`).

---

## 2.  Тест-кейсы

## Блок А: Авторизация и Безопасность (Security)
**Приоритет:** Critical  
**Техники:** EP (Классы эквивалентности), BVA (Граничные значения)

| ID | Описание (Что? Где? Когда?) | Шаги / Условия | Тестовые данные | Ожидаемый результат |
| :--- | :--- | :--- | :--- | :--- |
| **TC-SEC-01** | Доступ с валидным токеном (16 симв.) | Выполнить запрос к существующему продукту | `authToken: a1b2c3d4e5f6g7h8` | 200 OK |
| **TC-SEC-02** | Токен длиной 15 симв. (BVA) | Отправить запрос с токеном короче 16 симв. | `authToken: 123456789012345` | 401 Unauthorized |
| **TC-SEC-03** | Токен со спецсимволами (EP) | Отправить запрос с токеном, содержащим ! | `authToken: password!!123456` | 401 Unauthorized |
| **TC-SEC-04** | Отсутствующий токен | Выполнить запрос без параметра authToken | `authToken: null` | 401 Unauthorized |

## Блок B: Поиск ресурсов (Product ID)
**Приоритет:** High  

| ID              | Описание | Предусловия | Тестовые данные | Ожидаемый результат |
|:----------------| :--- | :--- | :--- | :--- |
| **TC-ID-01**    | Существующий продукт | В БД есть продукт с ID 1001 | `productId`: 1001 | 200 OK |
| **TC-ID-02**    | Несуществующий продукт | В БД нет продукта с ID 99999 | `productId`: 99999 | 404 Not Found |

## Блок C: Функциональное тестирование параметров (Positive)
**Приоритет:** Major  
**Техника:** Decision Table (Параметры не коррелируют между собой)
**Примечание:** Во всех запросах ниже подразумевается передача валидного authToken.

| ID | Комбинация параметров | Query String | Ожидаемый результат | Постусловия |
| :--- | :--- | :--- | :--- | :--- |
| **TC-FUNC-01** | Сет 1: true + Создатель + С-З | `?recalculate=true&owner=Создатель&region=Северо-Запад` | 200 OK, productStatus: 1 | Удалить данные |
| **TC-FUNC-02** | Сет 2: false + Пользователь + Сибирь | `?recalculate=false&owner=Пользователь&region=Сибирь` | 200 OK, productStatus: 0 | Удалить данные |
| **TC-FUNC-03** | Сет 3: null + null + Поволжье | `?recalculate=null&owner=null&region=Поволжье` | 200 OK | Удалить данные |
| **TC-FUNC-04** | Только обязательные поля | `?authToken=a1b2c3d4e5f6g7h8` (остальные опущены) | 200 OK | — |
